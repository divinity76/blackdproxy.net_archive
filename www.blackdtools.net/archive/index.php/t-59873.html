Hum, try doing it conditional events instead ... Here's a great script from @Malmis:

http://www.blackdtools.net/showthread.php?57339-Better-Mage-Exori-Exori-Gran

For your case you should edit it like so:



1
#number=#
1
0
0
0
exiva _strike = no
$nameofhex-id:{$hex-lastattackedid$}$
#string=#
earth elemental
0
0
0
exiva _strike = flam
1
#number=#
1
0
0
0
exiva _live2strike = 0
$hpofhex-id:{$hex-lastattackedid$}$
#number>#
10
0
0
0
exiva _live2strike = $numericalexp:{$_live2strike$}+1$
$hpofhex-id:{$hex-lastattackedid$}$
#number<#
100
0
0
0
exiva _live2strike = $numericalexp:{$_live2strike$}+1$
$_strike$
#string<>#
no
0
0
0
exiva _live2strike = $numericalexp:{$_live2strike$}+1$
$_live2strike$
#number=#
3
0
1000
0
exori $_strike$


earth elemental = creature name
flam = spell
10 = HP of creature, casts spell on it if HP is higher than that amount, useful if you're using a wand.

But please be careful! @Malmis stated in one of his threads that this might spam when out of mana or if creature isn't in Line-Of-Sight.

May spam when out of mana or attacking mobs not in Line-of-Sight, so might be unsafe. Use with caution.

Source: http://www.blackdtools.net/showthread.php?57391-Malmis-Scripts&highlight=exori
