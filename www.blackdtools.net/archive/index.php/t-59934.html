The cavebot seem to be getting confused when trying to leave ank, it goes up the ramp/stairs to leave but does not go back down the other side.

Any tips on what i should do?

maybe one of the floor changers is not recognized as floor changer. in that case the cavebot won't attempt to use in that way.

Solution:
- inspect the tile id of the floor changer not being used
- close blackd proxy
- add tile id to the list of floor changers in config-latestversion\config.ini
- reload blackd proxy and test cavebot now

-OR- just send me (by email) a screenshot of the stairs giving you problem plus the x,y,z position of such stairs.
you can get current x,y,z using the command
exiva pos
then I will visit the place and I will fix the problem for the next version of Blackd Proxy.
