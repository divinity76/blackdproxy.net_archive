They use a custom client with a custom login protocl -

you must reverse-engineer the login protocl used by highexp.eu then patch the blackd proxy source code to match the custom protocol - this may be very difficult, not for the faint of heart, so to speak :p

- if you're lucky though, they merely changed the encryption keys. that's easy to patch, and indeed they use a custom RSA encryption key, "13146904782324295394727183831500531303872348235911 60904321850610994950530421522584568082718058011218 77260432000307441697122963935205403345798421436352 15511866558075331708844673356368689217573532925210 92185242349394202170459886069740561271111457992918 31491301065152044360540623060663087830272779778816 065034131"

make a custom BlackD with patching the OTserverRSAkey in https://github.com/blackdtools/Blackd-Proxy-CLASSIC/blob/master/modRSA.bas#L9

there is a compiling tutorial here http://www.blackdtools.net/showthread.php?62649-(Tutorial)-How-to-compile-BlackD-Proxy-from-source-(incl-Win-7-x64)
